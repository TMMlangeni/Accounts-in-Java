# Accounts-in-Java
Banking System - Java OOP ImplementationA robust Java-based banking application demonstrating core Object-Oriented Programming (OOP) principles. This project manages different account types, implements custom exception handling for data integrity, and uses polymorphic behavior to calculate interest rates.🚀 Key FeaturesInheritance Hierarchy: Utilizes a base Account class with specialized Savings and Credit subclasses.Custom Exception Handling: Implements InvalidAccountNumberException to enforce business rules (e.g., 10-digit account number validation).Polymorphic Interest Calculation: Different logic for interest accrual based on account type and balance thresholds.Encapsulation: Strict use of private fields with public getters and setters to ensure data security.🛠 Project StructureThe project is built with a clear separation of concerns:Class/InterfaceDescriptionAccountThe abstract base class containing common attributes like accountNumber and balance.SavingsA subclass for savings accounts with a minimumBalance requirement.CreditA subclass for credit accounts featuring overdraftLimit logic.CalculableAn interface defining the contract for interest-bearing accounts.InvalidAccountNumberExceptionA custom runtime exception for validation failures.💻 Logic BreakdownInterest Calculation RulesSavings: Interest is only earned if the balance is above the minimumBalance.Credit: * Balance below overdraftLimit: 4x interest rate.Balance between overdraftLimit and 0: 2x interest rate.Positive Balance: Standard interest rate.ValidationThe system ensures that all account numbers are exactly 10 digits long. If an invalid number is provided during instantiation or through setAccountNumber(), the system throws a custom InvalidAccountNumberException.⚙️ How to RunClone the repository:Bashgit clone https://github.com/TMMlangeni/Banking-System-OOP.git
Compile the source files:Bashjavac *.java
Run the main application:Bashjava Account
